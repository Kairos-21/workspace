## 3. 模块通用样式

所有功能模块使用统一的 `.module` 容器：

| 属性 | 值 |
|------|-----|
| 背景 | `rgba(255,255,255,0.85)` |

## 4. 待办事项模块（Todo）

**位置**：第一行左栏  
**文件**：`前端/js/todo.js`, `前端/js/todo-global.js`

### 4.1 模块头部

| 元素 | 说明 |

### 4.2 输入区

```
┌──────────────────────────────────────┬──────────┐
│  输入框（placeholder: "输入待办，      │ 优先级下拉 │
│  回车添加，Tab切换优先级..."）          │          │

```

| 控件 | 类型 | 说明 |
|------|------|------|
| 文本输入框 | `<input type="text">` | flex: 1，回车添加待办，Tab 切换优先级 |
| 优先级下拉 | `<select>` | 选项：中优先级(默认) / 高优先级 / 低优先级 |

### 4.3 待办列表

每项待办（`.todo-item`）结构：

```
┌──────────────────────────────────────────────────┐
│ [复选框] [序号] [内容文字]          [操作按钮组]   │
│          [优先级徽章] [拖拽手柄]                   │
│          ┌── 子任务列表 ──────────────────────┐   │
│          │ [复选框] [序号] [子任务内容] [操作]  │   │
│          │ [复选框] [序号] [子任务内容] [操作]  │   │
│          └───────────────────────────────────┘   │
│          [+ 添加子任务]                           │
└──────────────────────────────────────────────────┘
```

**状态与交互**：

| 状态 | 表现 |
|------|------|
| 默认 | 灰色背景 (`--bg-primary`)，hover 变深 |
| 已完成 | `opacity: 0.6`，内容文字删除线 |
| 选中（键盘导航） | 蓝色边框 `2px solid #4A90D9`，蓝色背景微调 |
| 拖拽中 | `opacity: 0.5`，背景 `#f0f0f0` |
| hover 显示操作按钮 | 默认 `opacity: 0`，hover → `opacity: 1` |

**优先级徽章**（彩色圆角标签）：

| 优先级 | 颜色 | 背景色 |
|--------|------|--------|
| 高 (high) | 白色文字 | `#E74C3C` 红色 |
| 中 (medium) | 白色文字 | `#F39C12` 橙色 |
| 低 (low) | 白色文字 | `#27AE60` 绿色 |
| 日常 (daily) | 白色文字 | `#3498DB` 蓝色 |

**键盘快捷键**：

| 操作 | 按键 |
|------|------|
| 添加待办 | 输入框内按 Enter |
| 切换优先级 | 输入框内按 Tab |
| 多行输入 | Ctrl + Enter |
| 选中待办 | 单击文字区域 |
| 切换选中项 | ↑ / ↓ |
| 移动排序 | Ctrl + ↑ / Ctrl + ↓ |
| 取消选中 | Esc |

**子任务系统**：
- 每个待办可包含多个子任务（subtasks）
- 子任务有独立复选框、序号、拖拽排序
- 子任务通过左侧 2px 灰色竖线缩进
- 子任务 ≥3 行时增加间距 (`has-many` class)
- "+ 添加子任务" 触发器默认半透明，hover 高亮
- 子任务输入框自动增高（`min-height: 1.6em`）

**日期导航**：
- 点击日历日期可跳转到对应日期的待办列表
- 显示文案："今天" / "昨天" / "YYYY年M月D日 周X"
- 日常待办（daily）每天自动重置为未完成
- 未完成的持续待办自动跨天继承

**列表最大高度**：300px，超出滚动

## 6. 番茄钟模块（Pomodoro）

**位置**：第一行右栏  
**文件**：`前端/js/pomodoro.js`

### 6.1 模块头部

| 元素 | 说明 |
|------|------|
| 标题 `🍅 番茄钟` | 固定文字 |
| `⚙️` 按钮 | 展开/收起设置面板 |

### 6.2 计时器主体

```
        ┌─────────────────────┐
        │    ╭─────────╮      │
        │   ╱           ╲     │  ← SVG 环形进度条（200×200 viewBox）
        │  │   25:00     │    │  ← 倒计时数字居中
        │   ╲           ╱     │
        │    ╰─────────╯      │
        └─────────────────────┘
                工作              ← 状态文字
          🍅 0 / 4              ← 完成数 / 目标数

        [▶ 开始] [⟳ 重置] [⏪ 补录]
        [⏩ 跳过休息]（仅在休息时显示）
```

**环形进度条规格**：
- 圆心 (100, 100)，半径 90
- 背景圆环：`#E0E0E0`，stroke-width 8
- 进度圆环：`#3498DB`（工作中）/ `#27AE60`（休息中），stroke-width 8，圆头端点
- 周长 `565.48`（= 2π×90），用 `stroke-dashoffset` 控制进度
- 旋转 -90° 使进度从 12 点方向开始
- 1 秒 linear 过渡动画

**计时器显示**：
- 等宽字体（Monaco / Consolas）
- 字号 2.2rem，font-weight 600
- 格式 `MM:SS`
- 内容不可选中

### 6.3 控制按钮

| 按钮 | 场景 | 行为 |
|------|------|------|
| `▶ 开始` | 初始/暂停状态 | 开始计时 |
| `⏸️ 暂停` | 运行中 | 暂停计时（按钮文字切换） |
| `⟳ 重置` | 始终显示 | 重置当前计时单元 |
| `⏪ 补录` | 始终显示 | 弹出输入框，填写已工作分钟数，从剩余时间扣减 |
| `⏩ 跳过休息` | 仅在休息阶段显示 | 跳过休息，直接开始下一工作单元 |

### 6.4 设置面板（默认隐藏）

展开后显示在计时器下方：

| 设置项 | 控件 | 默认值 | 范围 |
|--------|------|--------|------|
| 工作时长 | range 滑块 | 25 分钟 | 1-60 |
| 短休息 | range 滑块 | 5 分钟 | 0-30 |
| 长休息 | range 滑块 | 15 分钟 | 0-60 |
| 长休息间隔 | range 滑块 | 4 个番茄 | 2-8 |
| 目标个数 | range 滑块 | 4 个 | 1-16 |
| 声音提醒 | 复选框 | 开启 | — |
| 提示音音量 | range 滑块 | 80% | 0-100 |
| 提示音管理 | 声音列表 + 上传按钮 | 默认 "qing" 提示音 | 支持 MP3/WAV/OGG/M4A |

**声音列表**：列出所有可用提示音，点击选中，显示当前选中状态。支持上传自定义音频文件（最大 5MB）。

### 6.5 补录弹窗

点击 `⏪ 补录` → 弹出输入框：
- 输入已实际工作的分钟数（整数）
- 从当前剩余时间中扣除
- 剩余时间归零则自动完成当前番茄钟

### 6.6 状态与行为

| 场景 | 行为 |
|------|------|
| 页面关闭/隐藏 | 自动暂停，状态保存到 sessionStorage |
| 冷启动（新会话） | 恢复为暂停状态，等待手动继续 |
| 正常刷新 | 尝试恢复计时状态 |
| 后台标签页 | 基于 Date 时间戳精确计时，不受浏览器节流影响 |
| tick() | 每 5 秒保存一次进度 |
| 番茄完成 | 播放提示音，自动切换到休息/下一轮 |

## 7. 成长花园面板（Game System）

**位置**：Row 1 下方，可折叠  
**文件**：`前端/js/game-system.js`  
**默认状态**：隐藏（`display: none`），点击 Header 中 `🌱 Lv.X` 按钮展开

### 7.1 模块头部

| 元素 | 说明 |
|------|------|
| 标题 `🌱 成长花园` | 固定文字 |
| `▲` 按钮 | 收起面板 |

### 7.2 三栏布局

```
┌──────────────┬──────────────────┬──────────────┐
│   等级 & XP   │    今日任务       │  连胜 & 成就  │
├──────────────┼──────────────────┼──────────────┤
│ 🌿 Lv.X     │ 📋 今日任务       │     [7]      │
│ [████░░░░]  │ ✅ 完成3个待办     │   天连胜      │
│  50 / 100   │ ✅ 完成2个番茄钟   │              │
│              │ ⬜ 完成1个高优待办 │ 🏆 最新成就   │
│ 总待办: 42  │ ✨ 完成日常待办    │  🌱 🔥 📋    │
│ 总番茄: 18  │                  │              │
└──────────────┴──────────────────┴──────────────┘
```

### 7.3 左栏：等级 & XP

| 元素 | 说明 |
|------|------|
| 等级徽章 | 渐变绿色背景（`#a8e063 → #56ab2f`），白色大字 "🌿 Lv.X"，圆角 12px |
| 升级动画 | `levelUpBounce` 弹跳动画（0.6s，缩放到 1.18 再回弹） |
| XP 进度条 | 14px 高圆角条，绿色渐变填充，平滑过渡 0.5s |
| XP 文字 | 右下对齐小字 "X / Y XP" |
| 统计区 | 总待办完成数、总番茄完成数等（`.game-stat-item`） |

**等级公式**：Level N 所需累计 XP = `100 × N × (N+1) / 2`

### 7.4 中栏：今日任务

4 项每日任务，每项显示：

| 元素 | 说明 |
|------|------|
| 图标 | 任务 emoji |
| 名称 | 任务描述文字 |
| 进度 | "X / Y"（未完成灰色，已完成绿色） |
| 完成标记 | ✅ / ⬜ |

**4 项每日任务**：

| 任务 | 图标 | 目标 |
|------|------|------|
| 完成 N 个待办 | 📝 | 3 |
| 完成 N 个番茄钟 | 🍅 | 2 |
| 完成 1 个高优待办 | 🔥 | 1 |
| 完成日常待办（可缺1）| 🔄 | 至少完成1项，最多缺1项 |

**任务完成态**：`opacity: 0.55`，绿色淡背景，内容文字删除线，进度数字变绿。

**全部日常完成特效**：金色渐变背景 + 金色边框 + 发光呼吸动画（2s 交替循环）。

**完美一天判定**：4/4 全部完成（日常待办至少完成 1 项），触发连胜递增。

### 7.5 右栏：连胜 & 成就

**连胜显示**：
- 渐变粉金色背景（`#ffd89b → #f093fb`）
- 超大数字（2.8rem），白色带阴影
- "天连胜" 标签

**最新成就**：
- 圆形徽章 40×40px，灰色底，金色边框
- hover 放大 1.15 倍
- 最多显示最近 4-5 个

### 7.6 成就系统

**17 个成就**，分 4 类：

| 类别 | 成就 | 触发条件 |
|------|------|----------|

## 18. 关键交互模式

### 18.1 拖拽排序
- **待办事项**：HTML5 drag，拖拽手柄（⋮⋮），拖动时半透明
- **子任务**：HTML5 drag，拖拽手柄（⋮⋮），在子任务列表内排序
- **快捷方式**：HTML5 drag，在网格内排序 + 支持拖入文件夹
- **备忘录条目**：HTML5 drag，在备忘录内排序

### 18.2 键盘快捷键

| 快捷键 | 模块 | 行为 |
|--------|------|------|
| Enter | 待办输入框 | 添加待办 |
| Tab | 待办输入框 | 切换优先级 |
| Ctrl+Enter | 待办输入框 | 多行输入 |
| ↑ ↓ | 待办列表 | 切换选中项 |
| Ctrl+↑ Ctrl+↓ | 待办列表 | 移动选中项排序 |
| Esc | 待办列表 | 取消选中 |
| Esc | 弹窗 | 关闭弹窗 |

### 18.3 Hover 显示操作

此模式贯穿整个 UI：
- 待办条目 hover → 显示操作按钮（编辑/删除/拖拽）
- 子任务 hover → 显示删除按钮 + 拖拽手柄
- 快捷方式卡片 hover → 显示编辑/删除按钮
- 备忘条目 hover → 显示操作工具栏

### 18.4 自动保存

所有数据修改通过 `window.debouncedSave()` 触发，防抖延迟 ~500ms，优先级：
1. 写入 LocalStorage
2. 通过 `POST /api/data` 持久化到服务端 JSON 文件
